# Security Policy

## Supported Versions

The following versions of this project receive security updates.

| Version | Supported |
|----------|-----------|
| Latest | ✅ Yes |
| Older Versions | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

You can contact the project maintainer using GitHub Issues or by email.

Please include:

- Description of the vulnerability
- Steps to reproduce
- Expected behavior
- Screenshots (if applicable)
- Suggested fix (optional)

Security issues will be reviewed and addressed as soon as possible.

---

## Best Practices

This project follows these security recommendations:

- Never commit AWS Access Keys.
- Never commit SSH private keys.
- Store secrets using environment variables or a secrets manager.
- Use IAM roles with least privilege.
- Keep Docker images updated.
- Regularly patch EC2 instances.
- Enable Security Groups with minimum required ports.
- Restrict database access to private networks.
- Enable HTTPS in production environments.
